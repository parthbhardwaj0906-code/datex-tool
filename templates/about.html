DATEX

Technical Architecture & Specifications

Automated Data Integrity & Anomaly Detection Framework

01. Executive Overview

Datex is a deterministic data integrity and pre-processing engine designed to ingest tabular datasets, evaluate structural metrics, and flag statistical anomalies. By executing vectorized operations via Pandas backend routines, Datex reduces raw data audit latency and highlights missing data distributions and statistical outliers.

02. Data Ingestion & Schema Parsing

Upon user payload upload (via multi-part HTTP standard requests), the backend handles CSV stream parsing into structured DataFrame representations:

  • Type Inference & Vectorization: Automatically maps continuous numerical dimensions, categorical factors, and missing null indicators (NaN / None).
  • Null Value Auditing: Computes column-wise aggregated metrics for missing values, calculating total missing row counts and relative missingness proportions.

03. Outlier Detection Methodology

Datex leverages non-parametric statistical measures to identify extreme deviations without assuming normal Gaussian distributions across feature spaces:

  • Interquartile Range (IQR) Analysis: For continuous numerical columns, quartiles Q1 (25th percentile) and Q3 (75th percentile) are evaluated to calculate interquartile spread (IQR = Q3 - Q1).
  • Boundary Evaluation: Observations residing outside the dynamic interval [Q1 - 1.5 × IQR, Q3 + 1.5 × IQR] are flagged as anomalous data points.
  • Z-Score / Standardized Residual Analysis: Optional parametric scaling to measure feature deviation from the mean in standard units.
